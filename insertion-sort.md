# Дослідження алгоритму сортування вставкою (Insertion Sort)
**Сортування вставкою (Insertion Sort)** — це порівняльний алгоритм сортування, який будує фінальний відсортований масив по одному елементу за раз, шляхом ітеративного переміщення кожного елемента з невідсортованої частини на його правильну позицію у відсортованій частині масиву.
## Галузі математики, що використовуються для опису сортування вставкою (Insertion Sort)
### 1.Математичний аналіз (Асимптотичний аналіз складності) : 
Математичний аналіз використовується для оцінки поведінки алгоритму, коли розмір масиву прагне до нескінченності (N → ∞).

* **Нотація Big-O (О-нотація):** Використовується для встановлення верхньої, нижньої та точної меж зростання кількості операцій.
* **Найкращий випадок (Ω(N)):** Математична модель для вже відсортованого масиву. Кількість операцій описується лінійною функцією.
* **Найгірший випадок (O(N²)):** Модель для реверсного масиву. Кількість порівнянь перетворюється на суму арифметичної прогресії, що дає квадратичну функцію.

  *Математичний аналіз корисний для прогнозування масштабованості програми ще до запуску.*
### 2.Математична логіка (Теорія доведення та індукція)
Використовується для суворого доведення того, що алгоритм є коректним і гарантовано видасть правильний результат на будь-яких вхідних даних.

* **Поняття «Інваріант циклу» (Loop Invariant):** Це логічне твердження про стан даних, яке є істинним перед початком циклу, залишається істинним під час кожної ітерації та виконується після його завершення.
* **Метод математичної індукції:** Доведення коректності Insertion Sort розбиється на три класичні кроки індукції:<br>
 1.**Ініціалізація (база індукції):** Перед першим кроком (`i = 1`) підмасив A[0] складається з одного елемента й автоматично є відсортованим за визначенням.<br>
  2.**Збереження (крок індукції):** Якщо підмасив відсортований для кроку `k`, то після зсуву елементів та вставки поточного елемента на правильне місце, підмасив залишається відсортованим і для кроку `k + 1`.<br>
  3.**Завершення:** Коли лічильник досягає N, інваріант циклу свідчить, що весь масив A[0...N-1] тепер складається з початкових елементів, але у відсортованому порядку.

    *Математична логіка корисна гарантії коректності та надійності коду.*

### 3.Теорія ймовірностей та математична статистика
Використовується для аналізу алгоритму в середньому випадку (Average-case), коли наперед невідомо, у якому стані прийдуть дані.

* **Модель випадкової перестановки:** Припускається, що всі N! можливих перестановок елементів вхідного масиву є рівноймовірними.
* **Математичне сподівання:** Обчислюється математичне сподівання кількості порівнянь та копіювань. На основі ймовірнісного аналізу доводиться, що в середньому елемент зсується на половину довжини вже відсортованої частини, що підтверджує середню часову складність як **Θ(N²)**.

    *Теорія ймовірностей та математична статистика корисна для оцінки ефективності алгоритму на практиці при обробці випадкових вхідних даних.*

## Реалізація алгоритму  алгоритму сортування вставкою (Insertion Sort):
### Реалізація за допомогою C++ : 
```cpp
#include <iostream>
using namespace std;

void insertionSort(int arr[], int n)
{
    for (int i = 1; i < n; ++i) {
        int key = arr[i];
        int j = i - 1;

        /* Зсуваємо елементи масиву arr[0..i-1], які є
           більшими за key, на одну позицію вперед 
           від їхнього поточного положення */
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j = j - 1;
        }
        arr[j + 1] = key; // Вставляємо ключ на його правильне місце
    }
}

void printArray(int arr[], int n)
{
    for (int i = 0; i < n; ++i)
        cout << arr[i] << " ";
    cout << endl;
}

int main()
{
    int arr[] = { 156, 24, 13, 90, 32, 2, 25, 52, 67 };
    int n = sizeof(arr) / sizeof(arr[0]);

    insertionSort(arr, n);
    printArray(arr, n);

    return 0;
}
```
### Вхідні дані, які обробляє алгоритм :

  1. Одновимірний статичний масив цілих чисел(`int arr[]`).
  2. Ціле число  (`int n`), що визначає точну кількість елементів у масиві.
* **Початковий стан:** `int arr[] = { 156, 24, 13, 90, 32, 2, 25, 52, 67 }` (Елементи масиву розташовані в недетермінованому (несортованому) порядку).

  ### Додаткові обмеження та властивості

* **Просторова складність:** Строго $O(1)$. Алгоритм є просторово ефективним, оскільки створює лише дві додаткові локальні змінні в стеку (`key` та `j`). Обсяг використаної пам'яті є константним і не залежить від розміру масиву $n$.

* **Часова складність:** Аналіз часової складності базується на класичній методології оцінки алгоритмів, де кількість операцій чітко розмежовується залежно від структури та початкового стану вхідних даних (найкращий, середній та найгірший випадки):
  
  * **Найгірший випадок (Worst-case): $O(n^2)$** — Цей сценарій виникає, коли вхідний масив уже відсортований, але у зворотному порядку (від найбільшого до найменшого). У такому разі кожен новий елемент під час вставки змушений пройти крізь усю поточну відсортовану ліву частину масиву.  
    Математично кількість операцій порівняння та зсуву для $n$ елементів описується сумою арифметичної прогресії:
    $$1 + 2 + 3 + \dots + (n - 1) = \frac{n(n - 1)}{2} = \frac{n^2}{2} - \frac{n}{2}$$
    
    Для великих значень $n$ член $\frac{n^2}{2}$ домінує, тому в Big O нотації менш значущі члени та константні коефіцієнти відкидаються, що дає фінальну складність:
    $$O\left(\frac{n^2}{2}\right) = O\left(\frac{1}{2} \cdot n^2\right) = O(n^2)$$
    
    <img width="370" height="293" alt="image" src="https://github.com/user-attachments/assets/b1dc6d7a-9ff7-4272-bc71-1a9f52458193" />

  * **Найкращий випадок (Best-case): $\Omega(n)$** — Виникає тоді, коли масив із самого початку є повністю відсортованим. За цих умов внутрішній цикл `while` жодного разу не виконує своє тіло, оскільки умова `arr[j] > key` завжди хибна. Алгоритм здійснює лише $n - 1$ порівнянь у зовнішньому циклі `for`, що забезпечує лінійний час виконання.
  * **Середній випадок (Average-case): $\Theta(n^2)$** — Для випадкового (недетермінованого) розподілу чисел у масиві, кожен елемент у середньому переміщується на половину довжини вже відсортованого підмасиву. Це зменшує кількість реальних операцій удвічі порівняно з найгіршим випадком, проте асимптотично закон росту функції залишається квадратичним — $n^2$.

* **Стабільність :** Алгоритм є **стабільним**. Строга умова порівняння `arr[j] > key` у внутрішньому циклі гарантує, що дубльовані за значенням елементи зберігають свій початковий відносний порядок після сортування.

## Практичні вимірювання ефективності алгоритму сортування вставкою (Insertion Sort)
Для експериментального підтвердження теоретичних оцінок складності алгоритму було використано програмний код на мові C++. Тестування проводилося на базі динамічного масиву розміром $N = 10\,000$ елементів типу `int`. Заміри часу виконання здійснювалися за допомогою високоточного системного таймера з бібліотеки `<chrono>` (`high_resolution_clock`).

### 1. Методика проведення експерименту

Ефективність алгоритму вимірювалася для трьох принципово різних типів початкових даних:
1. **Випадковий масив:** Елементи генерувалися хаотично у діапазоні від `0` до `99999` за допомогою функції `rand()`. Це відповідає середньому випадку часової складності  $\Theta(n^2)$.
2. **Впорядкований масив (Найкращий випадок):** Масив заповнювався числами за зростанням (від `0` до `9999`). Це дозволяє перевірити лінійну складність алгоритму $\Omega(n)$ та його адаптивність.
3. **Зворотний масив (Найгірший випадок):** Масив заповнювався числами у спадному порядку (від `10000` до `1`). Це змушує алгоритм виконувати максимальну кількість зсувів у пам'яті $O(n^2)$.

#### Релізація експерименту за допомогою C++
```cpp
#include <iostream>
#include <ctime>
#include <chrono>

using namespace std;
using namespace std::chrono;

void insertionSort(int array[], int size) {
    for (int i = 1; i < size; ++i) {
        int key = array[i];
        int j = i - 1;

        while (j >= 0 && array[j] > key) {
            array[j + 1] = array[j];
            j = j - 1;
        }
        array[j + 1] = key;
    }
}

int main() {
    const int ARRAY_SIZE = 10000;
    srand(time(0));

    // ВИПАДКОВИЙ МАСИВ (Середній випадок)

    int* randomArray = new int[ARRAY_SIZE];
    for (int i = 0; i < ARRAY_SIZE; ++i) {
        randomArray[i] = rand() % 100000;
    }

    auto startRandom = high_resolution_clock::now();
    insertionSort(randomArray, ARRAY_SIZE);
    auto stopRandom = high_resolution_clock::now();
    auto durationRandom = duration_cast<microseconds>(stopRandom - startRandom);

    delete[] randomArray;

    // ВЖЕ ВПОРЯДКОВАНИЙ МАСИВ (Найкращий випадок)
    int* sortedArray = new int[ARRAY_SIZE];
    for (int i = 0; i < ARRAY_SIZE; ++i) {
        sortedArray[i] = i; // Числа йдуть по порядку: 0, 1, 2, 3...
    }

    auto startSorted = high_resolution_clock::now();
    insertionSort(sortedArray, ARRAY_SIZE);
    auto stopSorted = high_resolution_clock::now();
    auto durationSorted = duration_cast<microseconds>(stopSorted - startSorted);

    delete[] sortedArray;


    // ЗВОРОТНИЙ МАСИВ (Найгірший випадок)
    int* reversedArray = new int[ARRAY_SIZE];
    for (int i = 0; i < ARRAY_SIZE; ++i) {
        reversedArray[i] = ARRAY_SIZE - i; // Числа йдуть назад: 10000, 9999, 9998...
    }

    auto startReversed = high_resolution_clock::now();
    insertionSort(reversedArray, ARRAY_SIZE);
    auto stopReversed = high_resolution_clock::now();
    auto durationReversed = duration_cast<microseconds>(stopReversed - startReversed);

    delete[] reversedArray;

    cout << "=== Insertion Sort Test Results (N = " << ARRAY_SIZE << ") ===" << endl << endl;

    cout << "1. Random Array (Average Case):" << endl;
    cout << "   " << durationRandom.count() << " microseconds "
         << "(" << durationRandom.count() / 1000.0 << " milliseconds)" << endl << endl;

    cout << "2. Already Sorted Array (Best Case):" << endl;
    cout << "   " << durationSorted.count() << " microseconds "
         << "(" << durationSorted.count() / 1000.0 << " milliseconds)" << endl << endl;

    cout << "3. Reversed Array (Worst Case):" << endl;
    cout << "   " << durationReversed.count() << " microseconds "
         << "(" << durationReversed.count() / 1000.0 << " milliseconds)" << endl << endl;

    return 0;
}
```

### 3. Аналіз отриманих результатів

1. **Підтвердження адаптивності:** Практичний замір на **впорядкованому масиві** показав мінімальний час виконання (14 мілісекунд). Це доводить, що внутрішній цикл `while` у нашому коді жодного разу не виконував зсув елементів, оскільки умова `array[j] > key` завжди була хибною. Програма здійснила лише один лінійний прохід.
2. **Порівняння середнього та найгіршого випадків:** Час сортування **зворотного масиву** виявився приблизно вдвічі більшим, ніж для **випадкового**. Це ідеально збігається з математичною теорією: у випадковому масиві елемент у середньому зсувається лише на половину довжини відсортованої частини ($\frac{n^2}{4}$ операцій), тоді як у зворотному масиві — просувається до самого початку ($\frac{n^2}{2}$ операцій).
3. **Висновок для практичного використання:** Отримані мілісекунди демонструють, що для обробки $10\,000$ елементів  алгоритм сортування вставкою(Insertion Sort) справляється швидко завдяки високій частоті сучасного процесора. Проте, через параболічний ріст часу ($n^2$), збільшення масиву, наприклад, до $100\,000$ елементів призведе до сповільнення роботи програми вже у 100 разів (час зросте до кількох секунд), що робить його неефективним для великих обсягів випадкових даних.

## Приклади використання алгоритму сортування вставкою (Insertion sort):
* **Сортування карт у руці :** Коли гравець бере карти з колоди по одній, він автоматично використовує сортування вставкою (Insertion Sort). Перша карта вже вважається «відсортованим підмасивом». Беручи кожну наступну карту (наприклад, Валета), гравець порівнює її з наявними справа наліво і вставляє між десяткою та Дамою.
* **Таблиця лідерів у локальній грі:** Якщо у грі на одному пристрої беруть участь 5–10 гравців, після кожного раунду їхні результати оновлюються. Оскільки масив невеликий, алгоритм сортування вставкою (Insertion Sort) миттєво переміщує гравця, який щойно набрав бали, на правильну позицію вгору по списку.

## Більш реалістичні приклади використання алгоритму сортування вставкою (Insertion Sort)

* **Потокове досортування даних (Online Sorting):** Застосовується в системах, де дані надходять послідовним потоком і вже є частково або повністю впорядкованими. Якщо через затримку мережі окремі елементи надходять із запізненням, алгоритм вставки дозволяє ефективно інтегрувати їх у вже існуючий масив. Оскільки основна частина структури вже відсортована, додавання нових елементів відбувається за мінімальну кількість операцій, близьку до лінійного часу.
* **Оптимізація гібридних алгоритмів (Hybrid Sorting):** Використовується як завершальний етап у складних алгоритмах сортування (таких як Quick Sort або Merge Sort). На великих обсягах даних високу ефективність демонструє рекурсія, проте на малих підмасивах (зазвичай менше 10–15 елементів) накладні витрати на рекурсію та виклики функцій починають перевищувати користь від алгоритму. У таких сценаріях рекурсію зупиняють і передають керування сортуванню вставкою, яке завдяки простоті своєї структури обробляє малі масиви швидше.
* **Комп'ютерна графіка:** Використовується для завдань рендерингу та впорядкування в режимі реального часу.
