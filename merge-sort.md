# Дослідження алгоритму сортування злиттям (Merge Sort)
**Сортування злиттям (Merge Sort)** - це стабільний рекурсивний алгоритм сортування порівняннями, що базується на парадигмі «розділяй і володарюй» (divide and conquer). Він працює шляхом рекурсивного ділення вхідного масиву на дві половини, рекурсивного сортування цих двох половин і, нарешті, об'єднання їх знову для отримання відсортованого масиву.

**Детальніше пояснення алгоритму:**

**1. Фаза поділу**

Алгоритм починається з багаторазового поділу вхідного масиву навпіл. Спочатку основний масив ділиться на дві рівні (або майже рівні) частини. Потім кожна з цих частин ще більше ділиться на дві. Цей поділ продовжується рекурсивно, доки ми не досягнемо підмасивів, що містять лише один елемент. За визначенням, масив з одним елементом вже відсортований.

**2. Фаза злиття**

Після того, як у нас є наші одноелементні підмасиви, ми об'єднуємо їх відсортованим чином. Процес об'єднання порівнює елементи з двох відсортованих підмасивів і будує з них новий, більший відсортований масив. Для кожної пари відбувається:

* Порівняння перших елементів обох масивів.
* Вибір меншого елемента і додаванння його до нашого результуючого масиву.
* Перехід до наступного елемента масиву, з якого було взято менший елемент.
Повторення, доки не будуть оброблені всі елементи з обох масивів.

**3. Фаза комбінування**

У міру того, як об'єднання продовжується, ми просуваємося назад по дереву рекурсії. Кожна операція об'єднання створює все більші й більші відсортовані підмасиви. Те, що починається як об'єднання пар окремих елементів, переростає в об'єднання масивів з 2 елементів, потім з 4, потім з 8 і так далі. Кожен результат об'єднання зберігає свою властивість сортування, гарантуючи, що до моменту досягнення остаточного об'єднання у нас буде повністю відсортований масив.

## Галузі математики, що використовуються для опису сортування вставкою (Merge Sort)
### 1. Теорія алгоритмів та математичний аналіз (Асимптотичний аналіз) : 
Математичний аналіз та теорія алгоритмів застосовуються для оцінки ефективності алгоритму, коли розмір вхідного масиву $n$ прямує до нескінченності ($n \to \infty$).

* **Нотація O-велике (O)**: визначає верхню межу часу роботи алгоритму.
* **Нотація $\Omega$-велике ($\Omega$):** задає нижню межу.
* **Нотація $\Theta$-велике ($\Theta$):** вказує на точну асимптотичну оцінку, коли верхня і нижня межі збігаються.

На відміну від Insertion Sort, де найкращий випадок описується лінійною функцією ($\Omega$) , а найгірший - квадратичною прогресією $O(n^2)$, для Merge Sort незалежно від початкового стану даних (масив уже відсортований, повністю хаотичний чи реверсний), структура обчислень не змінюється. Це доводить, що Merge Sort є стабільнішим за Insertion Sort.

### 2. Дискретна математика (Теорія графів) : 
У дискретній математиці процес роботи Merge Sort моделюється за допомогою спеціального виду графу - досконалого бінарного дерева (Strict Binary Tree), яке також називають деревом рекурсії.
Моделювання у вигляді графу допомагає візуалізувати та підрахувати сумарну роботу Merge Sort. Оскільки на кожному рівні дерева рекурсії виконується злиття елементів, загальна вартість операцій на одному рівні завжди дорівнює $O(n)$.

### 2. Математична логіка (Теорія графів) : 
Математична логіка дає інструменти для суворого теоретичного доведення того, що алгоритм працює безпомилково і гарантовано повертає правильний результат (впорядковану послідовність) для будь-яких можливих вхідних даних. Оскільки Merge Sort є рекурсивним алгоритмом, класичний метод «інваріанта циклу» (який використовувався для ітеративного Insertion Sort) тут замінюється на метод структурної математичної індукції за розміром масиву ($n$):

 1. **База індукції:** Розглядається мінімальний випадок, коли розмір масиву $n \le 1$. Алгоритм одразу завершує роботу, не виконуючи дій. Масив з одного елемента (або порожній) є відсортованим за визначенням, отже, твердження є істинним.
  2. **Індукційне припущення та крок індукції:** Припускається, що алгоритм працює бездоганно для будь-яких малих масивів розміром менше ніж $n$. Отже, дві рекурсивно викликані половини розміром $\lfloor n/2 \rfloor$ та $\lceil n/2 \rceil$ повертаються повністю впорядкованими. Далі логічно доводиться коректність допоміжної процедури злиття (Merge): якщо на вхід подано два відсортовані блоки, то покрокове порівняння їхніх елементів гарантовано дасть один загальний правильний масив розміром $n$.
  3. **Завершення:** На основі кроків 1 та 2 індукція доводить, що після завершення фінального рівня рекурсії весь початковий масив буде строго впорядкованим за зростанням.

## Реалізація алгоритму  алгоритму сортування злиттям (Merge Sort):
### Реалізація за допомогою C++ : 
```cpp
#include <iostream>
#include <vector>
using namespace std;

// Об'єднує два підмасиви arr[].
// Перший підмасив - arr[left..mid]
// Другий підмасив arr[mid+1..right]
void merge(vector<int>& arr, int left, 
                     int mid, int right){
                         
    int n1 = mid - left + 1;
    int n2 = right - mid;

    vector<int> L(n1), R(n2);

    for (int i = 0; i < n1; i++)
        L[i] = arr[left + i];
    for (int j = 0; j < n2; j++)
        R[j] = arr[mid + 1 + j];

    int i = 0, j = 0;
    int k = left;

    // Об'єднати тимчасові вектори назад в arr[left..right]
    while (i < n1 && j < n2) {
        if (L[i] <= R[j]) {
            arr[k] = L[i];
            i++;
        }
        else {
            arr[k] = R[j];
            j++;
        }
        k++;
    }

    // Скопіювати решту елементів L[](якщо такі є)
    while (i < n1) {
        arr[k] = L[i];
        i++;
        k++;
    }

    // Скопіювати решту елементів R[](якщо такі є)
    while (j < n2) {
        arr[k] = R[j];
        j++;
        k++;
    }
}

void mergeSort(vector<int>& arr, int left, int right){
    
    if (left >= right)
        return;

    int mid = left + (right - left) / 2;
    mergeSort(arr, left, mid);
    mergeSort(arr, mid + 1, right);
    merge(arr, left, mid, right);
}


int main(){
    
    vector<int> arr = {70, 30, 50, 10};
    int n = arr.size();

    mergeSort(arr, 0, n - 1);
    for (int i = 0; i < arr.size(); i++)
        cout << arr[i] << " ";
    cout << endl;
    
    return 0;
}
```

  1. Динамічний масив цілих чисел (вектор) (`std::vector<int> arr`).
  2. Межі сортування: цілі числа (`int left та int right`), які визначають індекси початку та кінця конкретного  підмасиву, що обробляється алгоритмом на поточному етапі рекурсії.

 ### Додаткові обмеження та властивості

* **Часова складність:**  Сортування злиттям стабільно виконується із часовою складністю $O(n \log n)$. Це робить його значно ефективнішим за такі алгоритми, як бульбашкове сортування (Bubble Sort) або сортування вставками (Insertion Sort), які в найгіршому випадку можуть виконуватися за квадратичний час $O(n^2)$.
  
* <img width="582" height="464" alt="image" src="https://github.com/user-attachments/assets/70bcfcbf-a23b-4167-868f-9693d1b0ae36" />



    * **Найкращий випадок ( $O(n  \log  n)$ ):** Навіть якщо вхідні дані вже повністю відсортовані, сортування злиттям усе одно чітко дотримується концепції «розділяй і володарюй», рекурсивно розбиваючи масив на частини та об'єднуючи їх назад. Кількість операцій не зменшується, що й зумовлює складність $O(n \log n)$.

    * **Середній випадок ( $\Theta(n \log n)$ ):** Незалежно від початкового порядку елементів, алгоритм завжди ділить масив навпіл і зливає отримані частини у строго структурований спосіб. Оскільки цей процес повторюється на $\log n$ рівнях рекурсії, а кожен рівень вимагає $O(n)$ операцій для злиття елементів, загальна складність залишається рівною $O(n \log n)$.

    * **Найгірший випадок ( $O(n  \log  n)$ ):** Коли вхідні дані впорядковані найгіршим чинником (наприклад, у зворотному порядку), сортування злиттям усе одно виконує той самий фіксований обсяг роботи. На відміну від швидкого сортування (Quick Sort), ефективність якого в найгіршому випадку може впасти до $O(n^2)$, сортування злиттям гарантує стабільну продуктивність $O(n \log n)$.

 
* **Просторова складність:** Хоча сортування злиттям ефективне з точки зору часу, воно пов'язане з додатковими витратами простору. Алгоритм вимагає додаткової пам'яті для зберігання розділених підмасивів під час об'єднання, що призводить до просторової складності O(n). Цей додатковий простір зберігає тимчасові масиви під час об'єднання елементів.


  Для рекурсивних реалізацій сортування злиттям також має O(log n)накладні витрати місця в стеку через рекурсивні виклики функцій. Однак, основне використання простору відбувається через допоміжні масиви, необхідні під час злиття. Це робить сортування злиттям менш зручним для середовищ з обмеженою пам'яттю порівняно з алгоритмами сортування "на місці" (in-place) , такими як швидке сортування (Quick Sort) або сортування купою (Heap Sort).

* **Стабільність :** Алгоритм сортування злиттям є **стабільним**, адже у функції `merge` під час порівняння елементів використовується умова `if (L[i] <= R[j])`. Знак `<=` гарантує: якщо елементи однакові, алгоритм завжди першим забере елемент із лівого підмасиву `L`, тобто той, який початково стояв лівіше (раніше). Це критично важливо, якщо у майбутньому сортуватимуться складні об'єкти (наприклад, список студентів спочатку за рейтингом, а потім за прізвищем).

* **Ефективність для зв'язаних списків :** Хоча для векторів та масивів алгоритм потребує $O(n)$ додаткової пам'яті, при сортуванні структур типу зв'язаний список сортування злиттям (Merge Sort) може працювати за $O(1)$ додаткової пам'яті. Це відбувається, через те, що у списках елементи не лежать у пам'яті послідовно, а зв'язані вказівниками і процедуру злиття можна виконати, просто змінюючи напрямок вказівників, без створення тимчасових копій елементів. Тобто для списків Merge Sort стає In-place алгоритмом, зберігаючи часову складність $O(n \log n)$.

## Практичні вимірювання ефективності алгоритму сортування злиттям (Merge Sort)

```cpp
#include <iostream>
#include <vector>
#include <ctime>
#include <chrono>
#include <fstream>

using namespace std;
using namespace std::chrono;


void merge(vector<int>& arr, int left, int mid, int right) {
    int n1 = mid - left + 1;
    int n2 = right - mid;

    vector<int> L(n1);
    vector<int> R(n2);

    for (int i = 0; i < n1; i++) L[i] = arr[left + i];
    for (int j = 0; j < n2; j++) R[j] = arr[mid + 1 + j];

    int i = 0, j = 0, k = left;

    while (i < n1 && j < n2) {
        if (L[i] <= R[j]) {
            arr[k] = L[i];
            i++;
        } else {
            arr[k] = R[j];
            j++;
        }
        k++;
    }

    while (i < n1) {
        arr[k] = L[i];
        i++;
        k++;
    }

    while (j < n2) {
        arr[k] = R[j];
        j++;
        k++;
    }
}

void mergeSort(vector<int>& arr, int left, int right) {
    if (left < right) {
        int mid = left + (right - left) / 2;

        mergeSort(arr, left, mid);
        mergeSort(arr, mid + 1, right);

        merge(arr, left, mid, right);
    }
}

int main() {
    const int ARRAY_SIZE = 10000;
    srand(time(0));

    // 1. ВИПАДКОВИЙ МАСИВ (Середній випадок)
    vector<int> randomArray(ARRAY_SIZE);
    for (int i = 0; i < ARRAY_SIZE; ++i) {
        randomArray[i] = rand() % 100000;
    }

    auto startRandom = high_resolution_clock::now();
    mergeSort(randomArray, 0, ARRAY_SIZE - 1);
    auto stopRandom = high_resolution_clock::now();
    auto durationRandom = duration_cast<microseconds>(stopRandom - startRandom);

    // 2. ВЖЕ ВПОРЯДКОВАНИЙ МАСИВ (Найкращий випадок)
    vector<int> sortedArray(ARRAY_SIZE);
    for (int i = 0; i < ARRAY_SIZE; ++i) {
        sortedArray[i] = i;
    }

    auto startSorted = high_resolution_clock::now();
    mergeSort(sortedArray, 0, ARRAY_SIZE - 1);
    auto stopSorted = high_resolution_clock::now();
    auto durationSorted = duration_cast<microseconds>(stopSorted - startSorted);

    // 3. ЗВОРОТНИЙ МАСИВ (Найгірший випадок)
    vector<int> reversedArray(ARRAY_SIZE);
    for (int i = 0; i < ARRAY_SIZE; ++i) {
        reversedArray[i] = ARRAY_SIZE - i;
    }

    auto startReversed = high_resolution_clock::now();
    mergeSort(reversedArray, 0, ARRAY_SIZE - 1);
    auto stopReversed = high_resolution_clock::now();
    auto durationReversed = duration_cast<microseconds>(stopReversed - startReversed);

    // Запис результатів експерименту у CSV файл
    ofstream csvFile("merge_sort_results.csv");
    if (csvFile.is_open()) {
        csvFile << "Array_Size,Best_Case_Mks,Average_Case_Mks,Worst_Case_Mks\n";
        csvFile << ARRAY_SIZE << ","
                << durationSorted.count() << ","
                << durationRandom.count() << ","
                << durationReversed.count() << "\n";
        csvFile.close();
    }

    cout << "=== Merge Sort Test Results (N = " << ARRAY_SIZE << ") ===" << endl << endl;

    cout << "1. Random Array (Average Case):" << endl;
    cout << "   " << durationRandom.count() << " microseconds "
         << "(" << durationRandom.count() / 1000.0 << " milliseconds)" << endl << endl;

    cout << "2. Already Sorted Array (Best Case):" << endl;
    cout << "   " << durationSorted.count() << " microseconds "
         << "(" << durationSorted.count() / 1000.0 << " milliseconds)" << endl << endl;

    cout << "3. Reversed Array (Worst Case):" << endl;
    cout << "   " << durationReversed.count() << " microseconds "
         << "(" << durationReversed.count() / 1000.0 << " milliseconds)" << endl << endl;

    return 0;
}
```
